### Highest-Risk Patterns

1. **Silent data loss** (D4, C2, C4, C19): Multiple paths where user data (notes, messages, conversations) is lost with zero user feedback. Fire-and-forget async writes with no retry or fallback.

2. **Race conditions in client switching / message handling** (D2, C1, C3, C4): Shared mutable state (`currentClient`, `currentSupabaseConvId`, `persistenceTurnCounter`) modified before async operations complete. Can cause data bleed between clients (dashboard) or conversation splitting (chat).

3. **Auth token exposure** (D9, D10): Refresh tokens in plaintext localStorage and password reset tokens in URL hashes. Any XSS vulnerability becomes a persistent auth compromise.

4. **Cascading dependency failures** (D6, D7, C5, C6): No timeouts, no partial failure handling, no circuit breakers. A single slow or failing Supabase/Botpress call can cascade into full UI lockup or forced logout.

5. **Broken mobile CSS** (D1): Extra closing brace in media query silently breaks all mobile responsive styles below that line.
